# `fw-commit` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Commit Path

Does it clearly describe commit, push, and PR flow?

### Readiness Gate

Does it check whether tests, review, and branch state are ready?

### Monitoring Notes

Does it include post-deploy monitoring or a grounded no-impact rationale?

### Browser Proof Discipline

When browser-visible, does it require or offer browser proof?

### Branch Safety

Does it protect the default branch?

### Commit Honesty

Does it preserve commit honesty and breaking-change checks?

### Spin Offer Discipline

After commit, does it offer `spin` only when a durable project lesson is
worth preserving?

### Hook-Gate Honesty

Does it describe commit or push checkpoints honestly for the active host,
including when a host can only warn instead of enforcing a confirm gate?

### Interaction Quality

Does it keep finish-stage choices structured and concise without raw-number
reply UX?
