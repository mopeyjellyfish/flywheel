# `fw-shape` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Mode Selection

Does `fw:shape` choose the correct internal shaping mode: ideate, brainstorm,
plan, or deepen?

### Artifact Discipline

Does the response describe or produce the right pre-work artifact without
drifting into implementation?

### Workflow Boundary

Does it keep `shape` as the first main workflow stage and preserve `work` as
the next main stage only after the shaping artifact is ready?

### Handoff Quality

Does it make the next handoff clear, including plan review and the deepen vs
work choice when planning is involved?

### Interaction Fit

When the shaping mode depends on a real ambiguity, does it ask one material
question or state the needed correction instead of silently guessing?
