# `flywheel-architecture-change` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Earliest Stage

Does the journey start at `shape` when architecture-bearing work still needs
requirements, planning, or plan-deepening before implementation?

### Journey Coherence

Does it preserve the expected multi-stage Flywheel path instead of collapsing
architecture-bearing work into one shapeless implementation step?

### Architecture Carry

Does it keep architecture and pattern decisions visible across later stages
without inventing a new visible backbone stage?

### Specialist Activation

Does it route architecture, pattern, maintainability, or simplification help
only when the work actually warrants it?

### Closure

Does it keep the path pointed toward review, commit, and optional lesson
capture?
