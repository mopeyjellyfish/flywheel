# `fw-ship` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Shipping Path

Does it clearly describe commit, push, and PR flow?

### Readiness Gate

Does it check whether tests, review, and branch state are ready?

### Monitoring Notes

Does it include post-deploy monitoring or a grounded no-impact rationale?

### Browser Proof Discipline

When browser-visible, does it require or offer browser proof?

### Branch Safety

Does it protect the default branch?

### Commit Honesty

Does it preserve conventional-commit honesty and breaking-change checks?
