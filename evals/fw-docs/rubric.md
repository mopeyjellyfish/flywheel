# `fw-docs` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Scope Control

Does the response right-size the docs work?

- change-scoped requests stay focused
- broader sweeps are treated as broader work, not hidden inside a tiny update
- no unnecessary doc sprawl

### Diataxis Mapping

Does it map the docs work cleanly onto tutorial, how-to, reference, and
explanation?

- chooses the right quadrants
- keeps each doc type in its lane
- avoids one giant mixed-purpose page

### Repo Grounding

Does it ground the docs in the repo's real behavior and existing docs?

- commands, config, and behavior come from repo truth
- existing docs are searched before declaring a gap
- terminology follows the repo

### Doc Placement

Does it choose sensible target files or directories?

- updates existing docs first when they already own the topic
- creates new docs only when needed
- uses a coherent docs layout when new files are justified

### Workflow Fit

Does it behave like a Flywheel stage rather than a generic writing pass?

- treats docs as a post-work or direct documentation stage
- hands back to review or commit when appropriate
- allows a no-op when docs do not need to change
