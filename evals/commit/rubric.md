# `conventional-commit` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Commit Shape

Does the response provide a real conventional commit header?

### Type Selection

Does it choose the expected commit type for the case?

### Breaking Change Gate

For breaking-change cases, does it require explicit user approval before using
`!` or `BREAKING CHANGE`?

### Output Shape

Does it provide a useful commit result, such as header plus optional body or
footer guidance?
