# `flywheel-handoff-gates` Grading Rubric

Score each dimension `0`, `1`, or `2`.

## Dimensions

### Handoff Card

Does the response close each material stage with a compact handoff containing
Stage, Artifact, Ready, Open decisions, Evidence, and Next?

### Readiness Gate

Does it check the relevant readiness gate before crossing into the next stage
instead of relying on vague confidence?

### Artifact Carry

Does it carry plans, review results, evidence, rollout notes, or solution notes
forward without depending on chat memory?

### Boundary Discipline

Does it preserve approval boundaries, especially plan approval before work,
review before commit, and conditional spin before final commit? Strong passes
use the host question tool when available for the plan-to-work approval gate
and do not treat a markdown menu as equivalent unless the tool is unavailable
or errors.

### Closure

Does it keep the flow pointed toward a reviewed branch and clean commit/PR
instead of stopping at an ambiguous handoff?
